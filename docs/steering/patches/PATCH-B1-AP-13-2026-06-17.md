## 3. Neue und geänderte Funktionen

### Neu in `app.js`

**`subtractMonths(yyyyMm, n)`**
YYYY-MM-Monatsarithmetik ohne `new Date()`. FwDateUtils bietet keine YYYY-MM-String-Subtraktionsfunktion — deshalb app-spezifisch implementiert (gemäß STATIONS_IMPLEMENTATION_PLAN §8 AP-13 Abbruchkriterium).

**`buildActiveJourneyWindow(msciData)`**
Leitet `latestMonth` aus dem letzten msciData-Eintrag ab, berechnet `startMonth = latestMonth − 119 Monate`. Gibt `{ startMonth, latestMonth, periodMonths: 120 }` zurück.

**`filterStationsForWindow(stations, window)`**
Filtert Stationen auf `startMonth <= station.date <= latestMonth`. Schließt `source_claimed_unchecked` still aus (kein Config-Error — Gate-Prüfung folgt in AP-14). `dynamic_latest_month` wird separat behandelt.

**`buildJourneyStations(stationsConfig, window)`**
Orchestrierung: historische Stationen filtern + `dynamic_latest_month`-Template auf `latestMonth` auflösen + Final-Reveal-Station ans Ende hängen. Keine Prioritätsauswahl, keine Gate-Prüfung (AP-14).

### Geändert in `app.js`

**`_loadDataImpl()`**
`latestMonth`/`startMonth` jetzt via `buildActiveJourneyWindow(msciData)` — explizite 119-Monats-Arithmetik statt Indexzugriff. `activeWindow` als eingefrorenes Feld in `appData` ergänzt.

**`buildAppContext(appData, monatlicheRate, startBetrag, stations)`**
Neuer vierter Parameter `stations`. Gibt jetzt `activeWindow` und `stations` im AppContext-Objekt zurück (statischer Kern gemäß APP_SPEC §13).

**`renderContent(container, appData, options, stationsConfig)`**
Berechnet `journeyStations = buildJourneyStations(stationsConfig, appData.activeWindow)` einmalig beim Init (nicht pro Slider-Event). Alle drei `buildAppContext`-Aufrufe mit `journeyStations` ergänzt.

## 4. Wie `latestMonth` und `startMonth` bestimmt werden

```
latestMonth = msciData[msciData.length - 1].date.slice(0, 7)
            → letzter valider CSV-Eintrag, Format YYYY-MM
            → kein new Date(), kein fixer Zeitraum

startMonth  = subtractMonths(latestMonth, 119)
            → explizite Monatsarithmetik
            → [startMonth … latestMonth] inklusiv = exakt 120 Monate
```

Beispiel für CSV mit letztem Eintrag 2025-04:
- `latestMonth` = `"2025-04"`
- `startMonth` = `"2015-05"` (8 + 108 + 4 = 120 Monate ✅)

## 5. Wie `dynamic_latest_month` behandelt wird

```js
const finalTemplate = stationsConfig.stations.find(s => s.date === 'dynamic_latest_month');
const finalRevealStation = Object.assign({}, finalTemplate, { date: window.latestMonth });
// → ans Ende der Journey gehängt, nie als Zwischenstation
```

- Wird nicht mehrfach eingebaut (AP-12-Validator stellt sicher: genau einmal in JSON)
- Kein hartcodiertes Datum — Wert kommt ausschließlich aus der CSV
